> 「机会永远存在，问题是你能不能看到，敢不敢抓住，能不能抓住。」

## 核心理念

**机会识别与筛选**是指系统性地发现商业机会，并用理性框架评估、筛选出值得投入的机会。搞钱的第一步是找到机会，但不是所有机会都值得追。

### 机会的本质

```
机会 = 需求 × 供给缺口 × 你的能力 × 时机

需求：有人愿意付费解决的问题
供给缺口：现有解决方案不够好或不存在
你的能力：你能提供解决方案
时机：现在做是合适的时候
```

## 发现机会的方法

### 一、从问题出发

```
问题即机会：

1. 自己的痛点
   「我遇到什么问题？」
   「有什么让我很烦的事？」
   「我愿意花钱解决什么？」

2. 身边人的痛点
   「朋友抱怨什么？」
   「同事在为什么发愁？」
   「家人需要什么帮助？」

3. 行业的痛点
   「这个行业什么最难做？」
   「什么环节效率最低？」
   「什么问题一直没人解决？」

核心：
有人愿意付费解决的问题 = 机会
抱怨但不愿付费 ≠ 机会
```

### 二、从变化出发

```
变化创造机会：

技术变化：
- 新技术出现（AI、区块链...）
- 技术成本下降
- 基础设施完善

政策变化：
- 新政策出台
- 监管放松/收紧
- 行业标准改变

人口变化：
- 老龄化
- 城市化
- 代际更替

消费变化：
- 消费升级/降级
- 新生活方式
- 新价值观

问自己：
「这个变化会创造什么新需求？」
「这个变化会让什么变得可能？」
「谁会从这个变化中受益/受损？」
```

### 三、从信息差出发

```
信息差即机会：

空间信息差：
- A地有，B地没有
- 国外有，国内没有
- 线上有，线下没有

时间信息差：
- 新趋势早期进入
- 别人不知道的信息
- 未来确定会发生的事

认知信息差：
- 你懂别人不懂的
- 你会别人不会的
- 你能别人不能的

获取信息差的方法：
- 多读、多看、多想
- 跨行业、跨领域学习
- 保持好奇心
- 接触不同圈子
```

### 四、从现有生意出发

```
改良式机会：

1. 做得更好
   现有方案体验差，我做得更好

2. 做得更便宜
   现有方案太贵，我成本更低

3. 做得更方便
   现有方案麻烦，我更简单

4. 做细分市场
   现有方案泛泛，我专注细分

5. 换个商业模式
   同样需求，不同变现方式

案例思考：
「这个行业的用户有什么不满？」
「这个产品为什么不够好？」
「如果重新做会怎么做？」
```

## 机会筛选框架

### 一、市场维度

```
市场检验清单：

规模：
□ 目标市场有多大？（TAM/SAM/SOM）
□ 能支撑多大的生意？
□ 值得投入吗？

增长：
□ 市场在增长还是萎缩？
□ 增长速度如何？
□ 增长驱动因素是什么？

竞争：
□ 现有竞争者有多少？
□ 竞争激烈程度如何？
□ 有没有巨头？

TAM（总市场）→ SAM（可服务市场）→ SOM（可获得市场）
不要高估市场规模
```

### 二、需求维度

```
需求检验清单：

真伪：
□ 是真需求还是伪需求？
□ 用户真的会为此付费吗？
□ 有没有验证过？

强度：
□ 是痛点还是痒点？
□ 是必须解决还是可有可无？
□ 用户愿意付多少钱？

频次：
□ 是高频需求还是低频需求？
□ 复购率会怎样？
□ 用户生命周期价值多少？

需求验证方法：
- 直接问潜在客户
- 预售测试
- MVP（最小可行产品）
- 落地页转化率测试
```

### 三、能力维度

```
能力匹配检验：

核心能力：
□ 我有做这件事的核心能力吗？
□ 如果没有，能获得吗？
□ 成本和时间是多少？

资源：
□ 需要什么资源？（钱、人、渠道）
□ 我有这些资源吗？
□ 能获取这些资源吗？

竞争力：
□ 我比别人强在哪里？
□ 我有什么独特优势？
□ 这个优势能持续吗？

团队：
□ 需要什么样的团队？
□ 能组建这样的团队吗？
□ 团队能力是否匹配？
```

### 四、时机维度

```
时机检验：

太早的风险：
- 市场教育成本高
- 基础设施不成熟
- 用户习惯没形成
- 可能成为先烈

太晚的风险：
- 市场已被占领
- 竞争激烈
- 用户已被锁定
- 利润空间小

判断时机：
□ 触发因素是否已经出现？
□ 早期用户是否已在行动？
□ 基础设施是否已经成熟？
□ 竞争格局是否已经固化？

最佳时机：
- 需求刚刚显现
- 技术刚刚成熟
- 竞争还不激烈
- 用户愿意尝试新方案
```

### 五、商业模式维度

```
商业模式检验：

盈利性：
□ 怎么赚钱？
□ 毛利率能有多少？
□ 能规模化盈利吗？

可持续性：
□ 有护城河吗？
□ 能持续赚钱吗？
□ 会被颠覆吗？

可扩展性：
□ 能规模化吗？
□ 边际成本如何？
□ 天花板在哪里？
```

## 避免机会陷阱

### 常见陷阱

```
1. 伪需求陷阱
   「我觉得这个需求很大」
   → 验证！让客户付费才算数

2. 自嗨陷阱
   「我的产品很好」
   → 好不好客户说了算

3. 市场幻觉
   「这个市场有万亿」
   → 和你有关的可能只有千万

4. 时机错判
   「现在是最好的时机」
   → 太早和太晚都是错

5. 能力高估
   「这个我能做」
   → 能做和做好是两回事

6. 竞争低估
   「没什么竞争」
   → 没看到不代表没有
```

### 反向检验

```
问自己：

1. 这个机会如果这么好，为什么别人没做？
   - 真的没人做？
   - 做了但失败了？
   - 有但你不知道？

2. 如果失败了，最可能是什么原因？
   - 需求不存在？
   - 时机不对？
   - 能力不够？
   - 资源不足？

3. 这个机会的最坏情况是什么？
   - 我能承受吗？
   - 怎么止损？
```

## 机会来源渠道

### 主动发现

```
1. 行业报告和数据
   - 投资机构报告
   - 行业研究报告
   - 政府统计数据

2. 一线信息
   - 与客户交流
   - 参加行业活动
   - 关注行业社群

3. 跨领域学习
   - 其他行业的做法
   - 国外的新模式
   - 相邻领域的创新

4. 趋势追踪
   - 技术趋势
   - 政策趋势
   - 社会趋势
```

### 被动接收

```
1. 朋友介绍
   - 保持人脉
   - 让别人知道你在找机会

2. 问题找上门
   - 有人问你能不能做某事
   - 有人愿意为某事付费

3. 意外发现
   - 做A发现B的机会
   - 副产品变成主产品
```

## 常见误区

| 误区 | 描述 | 纠正 |
|------|------|------|
| 等待完美机会 | 一直等最好的机会 | 好机会需要创造 |
| 追风口 | 什么热做什么 | 匹配自己的能力 |
| 忽视验证 | 想当然觉得行 | 快速低成本验证 |
| 贪多嚼不烂 | 什么机会都想要 | 聚焦最有把握的 |
| 只看机会不看风险 | 被机会冲昏头 | 机会和风险并看 |

## 刻意练习

### 日常练习

1. **机会雷达**：每周记录3个发现的机会
2. **变化追踪**：关注行业变化并思考机会
3. **需求观察**：记录身边人的抱怨和需求

### 进阶练习

1. **机会评估**：用评分卡评估一个机会
2. **验证实验**：设计低成本验证方法
3. **复盘学习**：研究成功/失败案例的机会判断
